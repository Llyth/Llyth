# 🚀 Test Technique DevOps – Helm Charts (http-echo & pgAdmin4)

## 📌 Contexte
Ce projet fait partie d’un test technique visant à démontrer la maîtrise de Kubernetes et Helm.  
Il contient deux applications déployées via des charts Helm :

1. **http-echo** : une application simple basée sur [mendhak/http-https-echo](https://hub.docker.com/r/mendhak/http-https-echo) qui renvoie les requêtes HTTP reçues.  
2. **pgAdmin4** : une interface web d’administration PostgreSQL, connectée à un service PostgreSQL externe.  

Ces déploiements mettent en avant les **bonnes pratiques Kubernetes** : probes, ressources, sécurité, persistance, ingress et scalabilité.

---

## 🏗️ Pré-requis
- Kubernetes (Kind utilisé pour ce test)  
- Helm v3  
- Ingress Controller (ex: [ingress-nginx](https://kubernetes.github.io/ingress-nginx/))  
- (Optionnel) PostgreSQL déployé dans le cluster  

---

## 🔧 Création du cluster Kind

Créer un cluster Kind multi-nœuds avec exposition des ports 80/443 pour l’ingress :

```yaml
# kind-config.yaml
kind: Cluster
apiVersion: kind.x-k8s.io/v1alpha4
name: cluster
nodes:
- role: control-plane
  extraPortMappings:
    - containerPort: 80
      hostPort: 80
      protocol: TCP
    - containerPort: 443
      hostPort: 443
      protocol: TCP
---
```

Commandes d'installation :

```
kind create cluster --config kind-config.yaml

```

## 🌐 Installation de l’ingress-nginx avec **kubectl** :

```
kubectl apply -f https://raw.githubusercontent.com/kubernetes/ingress-nginx/controller-v1.10.1/deploy/static/provider/kind/deploy.yaml

```

## 📦 Déploiement des applications

```
helm install http-echo ./http-echo -n demo --create-namespace

```
Accès via ingress :

```
URL : http://echo.localtest.me

```

```
helm install pgadmin4 ../pgadmin4 -n demo

```

```
URL : http://pgadmin.localtest.me
```
